query = '''

SELECT
    p.product_id,
    p.product_name,
    p.category_id,
    c.category_name,
    SUM(od.quantity) AS total_quantity,
    SUM(od.unit_price * od.quantity * (1 - od.discount)) AS total_sales,
    AVG(od.unit_price) AS avg_price
FROM
    order_details od
LEFT JOIN
    products p ON od.product_id = p.product_id
LEFT JOIN
    categories c ON p.category_id = c.category_id
GROUP BY
    p.product_id, p.product_name, p.category_id, c.category_name
ORDER BY
    total_sales DESC;'''

2.3 Aykr Deer Tespiti (Outlier Detection) IQR Yntemi

Aadaki fonksiyon, belirli bir saysal stun zerinde IQR (Interquartile Range) yntemini kullanarak aykr deerleri tespit eder:

Fonksiyon: detect_outliers_iqr(data, column)

  • Q1 (1. eyrek): Verinin alt %25lik ksmn temsil eder.
  • Q3 (3. eyrek): Verinin alt %75lik ksmn temsil eder.
  • IQR (eyrekler Aras Aralk): Q3 - Q1 ile hesaplanr.
  • Alt ve st Snrlar:
    • Alt snr = Q1 - 1.5 * IQR
    • st snr = Q3 + 1.5 * IQR

Bu snrlarn dnda kalan deerler aykr deer (outlier) olarak kabul edilir ve filtrelenerek dndrlr.

Bu yntem, dalmn dnda kalan u deerleri belirlemek ve gerekirse bu deerleri analizden karmak ya da ayrca deerlendirmek iin kullanlr.

def detect_outliers_iqr(data, column):
    Q1 = data[column].quantile(0.25)
    Q3 = data[column].quantile(0.75)
    IQR = Q3 - Q1
    lower_bound = Q1 - 1.5 * IQR
    upper_bound = Q3 + 1.5 * IQR

    outliers = data[(data[column] < lower_bound) | (data[column] > upper_bound)]
    return outliers

3.2 Farkl Modeller Oluturma

Makine renmesi modellerini optimize etmek, doru snflandrma ve tahmin sonular elde etmek iin kritik bir admdr. Decision Tree, K-Nearest Neighbors (KNN) ve Logistic Regression gibi yaygn algoritmalar, farkl veri yaplar ve problemler iin eitli avantajlar sunmaktadr. Bu farkl yaklam, model performansn artrmak iin hiperparametre optimizasyonu ve uygun deerlendirme metrikleriyle desteklendiinde, daha salam ve gvenilir sonular elde etmemizi salar.

  • TRUE - POSITIVE () : Gerek pozitif olan doru pozitif tahmin etme.
  • TRUE - NEGATIVE () : Gerek negatif olan doru negatif tahmin etme.
  • FALSE - POSITIVE () : Gerek negatif olan yanllkla pozitif tahmin etme (Type I error)
  • FALSE - NEGATIVE () : Gerek pozitif olan yanllkla negatif tahmin etme (Type II error)
  • Accuracy (Doruluk): Tm snfladrmalarn dengeli dalm gsterdii durumlarda baar tespiti iin kullanlabilir.
    Doru Tahmin Says / Toplam Tahmin Says
  • Precision (Kesinlik): Modelin pozitif tahminlerinden ka tanesinin gerekten pozitif olduunu lmek iin kullanlmaktadr. zellikle yanl pozitiflerin maliyetli olduu durumlarda nemlidir.
    / ( + )
  • Recall (Duyarllk): Gerek pozitif rneklerin ne kadarn doru tahmin ettiimizi gstermekte kullanlmaktadr. Karlmamas gereken kritik durumlarda ncelie sahiptir.
    / ( + )

def preprocess_input(sale: SaleInput):
    df_input = pd.DataFrame([sale.dict()])

    categorical_columns = ['product_name', 'category_name', 'yearquarter', 'city']
    df_input = pd.get_dummies(df_input, columns=categorical_columns, drop_first=True)

    for col in model_columns:
        if col not in df_input.columns:
            df_input[col] = 0

    return df_input[model_columns]
